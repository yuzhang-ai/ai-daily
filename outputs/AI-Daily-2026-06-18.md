# AI Daily 2026-06-18

## 一、今日最值得关注的 3 条内容

### 1. Agentic coding and persistent returns to expertise
- 来源链接：[Anthropic](https://www.anthropic.com/research/claude-code-expertise)
- 类型：研究报告 / 博客
- 推荐指数：5 星
- 一句话总结：Anthropic 分析约 40 万个 Claude Code 会话，结论是 Agent 越强，越奖励“懂业务、会判断”的人。
- 核心观点 3 条：
  1. 人主要决定 what to do，Agent 主要决定 how to do it。
  2. 领域专家每次指令能触发更多 Agent actions，成功率也更高。
  3. 成功要看 passing tests、commit、PR 等可验证证据，不能只看主观满意。
- 对 AI 产品经理 / FDE 的启发：作品集不要只展示“我会用 AI 写代码”，而要展示你能定义业务目标、拆验收标准、看懂失败日志、推动 Agent 交付可验证结果。
- 是否值得精读：是。

### 2. Agents' Last Exam
- 来源链接：[官网](https://agents-last-exam.org/) / [arXiv](https://arxiv.org/html/2606.05405v1)
- 类型：论文 / 基准评测
- 推荐指数：5 星
- 一句话总结：ALE 用真实职业工作流、GUI/CLI 环境和可验证产物来评估 Agent，而不是只做静态问答。
- 核心观点 3 条：
  1. 任务覆盖 55 个子行业、1.5K+ workflow，目标是经济价值而非刷榜。
  2. 评价以 output artifact 为核心：文件、表格、CAD、截图、代码、数值结果。
  3. 尽量不用泛化 LLM-as-judge，优先用 deterministic check 和硬门槛。
- 对 AI 产品经理 / FDE 的启发：做客户运营 / 企微 Agent 时，也要把“是否真正完成动作”落到可验证产物：标签是否写入、消息是否待人工确认、链接是否可打开、是否误触达。
- 是否值得精读：是。

### 3. Introducing Devin Desktop
- 来源链接：[Cognition](https://cognition.ai/blog/introducing-devin-desktop)
- 类型：博客 / 产品更新
- 推荐指数：4 星
- 一句话总结：Windsurf 演进为 Devin Desktop，核心信号是 IDE 正在从编辑器变成 Agent Command Center。
- 核心观点 3 条：
  1. 工程师工作流从 pair programming 转向管理多个本地和云端 agents。
  2. Spaces 让相关 agents 共享 context，适合复杂任务的并行推进。
  3. ACP 支持多 agent 接入，产品层竞争点转向统一调度、审查和上下文管理。
- 对 AI 产品经理 / FDE 的启发：未来 Agent 产品不只是 chat UI，而是任务队列、上下文仓库、权限控制、进度证据和人工验收面板。
- 是否值得精读：是。

## 二、今日最重要的一个观点

> 今天我最应该记住的是：Agent 的核心竞争力正在从“模型能不能回答”转向“系统能不能把真实工作流跑完、留下证据、接受人工验收”。AI PM / FDE 的价值不是替代工程师，而是把业务判断、上下文、工具权限、eval、风险控制组织成一个可复用 workflow。

## 三、值得深入学习的开源项目

- 项目名：Awesome LLM Apps
- GitHub 链接：[Shubhamsaboo/awesome-llm-apps](https://github.com/Shubhamsaboo/awesome-llm-apps)
- Star 数：114,887（GitHub API 2026-06-18 核验；页面约 115k）
- 解决什么问题：提供可运行的 AI Agent、RAG、MCP、multi-agent、always-on agent 模板。
- 为什么值得我学习：适合拆解“从 demo 到可交付应用”的目录结构、依赖、工具调用和 UI/后端组合。
- 如何用到我的项目 / 工作中：选一个客户运营场景，改造成“需求输入 - 资料检索 - 方案生成 - 风险检查 - 人工确认 - 企微触达草稿”的作品集项目。注意不要接入真实客户数据和密钥。
